
 정규화 사례 분석.



 # 제 1 정규형 First Normal Form.

1NF.
어떤 릴레이션 R에 속한 모든 도메인이 원자 값만으로 구성.

1NF 이상의 해결.
프로젝션으로 릴레이션을 분해( 부분 함수 종속을 제거 - 2NF )



 # 제 2 정규형 Second Normal Form.

2NF.
어떤 릴레이션 R이 1NF이고, 키에 속하지 않은 속성은 모두 기본 키에 완전 함수 종속.

2NF 이상의 해결.
프로젝션으로 릴레이션을 분해( 이행적 함수 종속을 제거 - 3NF )



 # 제 3 정규형 Third Normal Form.

3NF.
어떤 릴레이션 R이 2NF이고, 키에 속하지 않은 모든 속성이 이행적 함수 종속이 아닌 경우.

3NF 이상의 해결.
릴레이션 R의 결정자를 후보 키로 지정 - BCNF.



 # 보이스 / 코드 정규형 Boyce Codd Normal Form.

BCNF.
릴레이션 R의 모든 결정자가 후보키이다.



 # 제 4 정규형 Fourth Normal Formal.

4NF.
릴레이션 R에서 다치종속 A ->> B가 존재할 때 R의 모든 속성이 A에 함수 종속.

다치 종속 이상의 해결.
다치종속을 제거 -> 4NF 



 # 제 5 정규형 Fifth Normal Formal.

5NF.
릴레이션 R에 존재하는 모든 조인 종속이 릴레이션 R의 후보 키를 통에서만 성립.

조인 종속 이상의 해결.
조인 종속을 제거 -> 5NF.



 # 값이 여러 개인 정보 처리. ( 제 1 정규화 )

값이 여러 개인 정보 처리.
예를 들어 가족이라는 Data가 있을때 부모님과 형제들에 대한 정보처럼 여러 개 인 경우,
새로운 테이블을 만들어서 외래키로 참조하게 만들어서 사용한다.



 # 중복 되는 값 처리. 

중복 처리.
새로운 테이블을 만들어서 그룹화하여 상위 개체로 만들어준다.

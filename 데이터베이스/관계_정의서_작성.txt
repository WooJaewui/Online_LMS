 # 정규화 수행 프로세스.

제 1 정규화 수행.
소스 자료 분석 후 반복 그룹 속성 제거. ( 새로운 개체 생성 )
새로운 개체를 도출한 후 개체와 개체 간 관계 차수 파익 및 표기.

제 2 정규화 수행.
코드화 할 수 있는 속성을 파악. ( 부분 함수 종속 제거 )
코드화 대상 속성으로 구성되는 개체를 삭제하고, 새로운 개체를 생성.

제 3 정규화 수행.
이행 종속되는 속성을 제거. 
대부분의 업무는 제 3 정규화를 수행하면 정제된 모형이 도출됨.



 # 정규화에 따른 영향도 조사 및 조치.

데이터 모델링 특징.
정규화 규칙에 따라 데이터 모델링 수행.
논리적 데이터 모델링은 실전 프로젝트에서 주로 분석 단계 시 수행.
설계 단계 시에는 데이터베이스 성능을 고려해서 물리적인 데이터 모델링 수행.

반정규화 특징.
시스템의 성능 향상과 개발과 운영의 단순화를 위해 데이터 모델을 조정하는 프로세스.

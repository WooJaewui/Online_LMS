
 논리 데이터 모델의 품질 체크리스트.



 # 논리 데이터 모델의 관리 목적.

논리 데이터 모델의 정의.
개념 데이터 모델을 상세화하여 논리적인 데이터 집합( 개체 ), 관리항목( 속성 ), 관계를 정의한 모델.

관리 목적.
전체 데이터 구조에서 가장 핵심을 이루는 모델이다.
전체 업무 범위와 업무 구성 요소를 확인 가능.
데이터를 상세하게 정의 및 관리.
데이터 중복과 데이터의 불일치 방지.



 # 세부 관리 대상 및 방법.

세부 관리 대상 및 방법.
개체, 속성, 관계명을 표현하는 용어는 표준으로 정의.
현재의 업무가 구체적으로 반영.
데이터베이스의 객체는 변경하였으나 논리 데이터 모델에는 반영을 안하면 데이터 품질에 영향.
논리 데이터 모델의 변경 사항에 대한 이력도 관리( 권장 ).

주제영역.
업무 상 친밀도가 높은 데이터 집합을 하나의 주제영역으로 선언하여 관리.

주제영역 관리 기준.
원자성 : 하나의 단위 주제영역은 다른 주제영역의 개체나 관계의 영향을 받지 않는 개체 모임.
집중성 : 단위 주제영역 내의 개체와의 관계는 단위 주제영역 내에 집중.
업무지향성 : 업무적 명확성을 나타내는 단수 단위로 명명.

주제영역 관리 방법.
업무 상 동일한 영역으로 다루는 것이 효과적은 개체 집합들을 하나의 주제영역으로 선언.
업무의 다양성에 따라 여러 개로 나뉠 수 있음.
서로 다른 주제영역 간에 공유하는 개체의 수는 가급적 적어야한다.

개체.
개념 데이터 모델의 정의를 포함하고 보다 더 상세히 파악된 서브 타입 정보가 추가.

개체 관리 기준.
완전성 : 두 개 이상의 속성과 인스턴스( 값 )를 유지.
영속성 : 현재 관리하고 있는 데이터 집합 혹은 앞으로도 관리할 데이터 집합.
식별성 : 개체의 인스턴스를 개별적으로 구별할 수 있는 하나 이상의 속성이 존재.
동질성 : 하나의 데이터 집합인 개체에는 동질의 데이터가 모인 데이터 집합.
정규화 : 일반적으로 제 3 정규화까지 정규화( 권장 )

개체 관리 방법.
논리적인 단위로 정확히 분할하여 선언하되 중복되게 선언되지 않도록 하는 것이 중요하다.

속성.
개체 내에서 관리하고자 하는 정보 항목들을 의미.

속성 관리 기준.
원자성 : 의미 있는 최소 단위까지 분할되어야 한다.
일관성 : 하나의 속성은 하나의 데이터 유형을 가져야된다.
무결성 : 참조되는 속성의 데이터는 참조하는 속성의 데이터와 일치. ( 참조 무결성 )
정보성 : 업무 내에서 의미 있는 범위 내에서 상세화의 수준이 결정.

속성 관리 방법.
사용자의 요구에 따라 무분별하게 증가될 우려가 있다.
기존의 정보에서 추출이 불가능할 경우 새로운 추가가 필요.

관계.
개체 간의 논리적인 관계를 나타낸 것.

관계 관리 기준.
선택성 : 필수와 선택으로 나뉘어질 수 있음. 0 or 1.
관계형태 : 1 : 1 , 1 : M , M : N의 관계.
관계명칭 : 관계와 관계 사이의 관계명을 이해할 수 있도록 정의한다.

관계 관리 방법.
부모와 자식 간의 관계명을 반드시 정의.
관계의 선택성이 있어야된다.



 # 품질 검토 기준 항목.

개체의 품질 검토 기준 항목.
개체명 : 표기법에 따라 썼는지, 업무 개념과 내용이 정확하게 반영.
개체 정의 : 요구사항을 충족한지, 서브타입을 사용하여 구체적으로 정의.
통합 수준 : 통합이 고려되어 있는지.
권한 : 권한을 정의하였는지.
발생빈도 : 데이터 저장 건수 / 빈도를 파악.
관계 : 다른 개체와 하나 이상의 관계를 가지고 있는지.
법규 준수 : 관련 법규를 잘 사용했는지.
요구사항 : 요구사항과 잘 매핑이 되었는지.

속성의 품질 검토 기준 항목.
속성명 : 표기법에 따라 썼는지.
속성정의 : 개체 성격에 맞는 속성이 도출되었는지.
속성설명 : 성격, 관리 목적 등을 설명하였는지.
속성유형 : 속성에 맞게 식별자가 정의되었는지.
식별자 정의 : 모든 식별자가 적절하게 파악되었는가.
법규준수 : 관련 법규를 잘 사용했는지.
도메인 정의 : 속성의 도메인은 일관성 있게 정의되었는지.

관계의 품질 검토 기준 항목.
관계명 : 표기법에 따라 썼는지.
관계정의 : 요구사항과 일치하게 정의 했는지.
관계설명 : 관계에 대한 설명이 잘 이루어졌는지.
관계 표현 : 표기법에 따라 정확하게 표현했는지.
식별자 상속 : 모든 관계 속성들의 출처가 명확하게 파악되었는지.
요구사항 : 요구사항 매핑이 수행되었는가.
참조무결성 : 참조무결성을 정의하였는지.

모델 전반의 품질 검토 기준 항목.
주제영역 적절성 : 주제영역의 구성은 적절한가.
논리 모델 상세화 : 정규화가 미흡한 부분이 존재하는지.
이력관리 : 이력관리 방법이 적절한지.

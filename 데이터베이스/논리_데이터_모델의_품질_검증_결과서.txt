 # 개체, 속성, 관계 사례 분석.

개체 사례 분석.
기본 키가 자료의 유일성을 보장하는지 여부 검토. ( 유일성 확보 )
기본 키가 효율적인지 검토 여부. ( 유일성 확보 또는 효율적인 키를 생성 )
자료의 발생 유형이 유사한 개체의 통합 여부 검토. ( 테이블 구성을 변경 )
통합 또는 분리되어야 하는 개체의 여부 검토. ( 테이블 구성을 변경 - 통합 / 분리 )
추가로 생성되어야 하는 개체 유무 검토. ( 테이블 추가 )
기본 키의 순서가 정의되었는지 여부 검토. ( 많이 사용하는 기본키를 먼저 정의 )

속성 사례 분석.
반정규화 된 속성이 식별되는지 여부 검토. ( 같은 이름의 속성은 속성명을 바꿔준다 )
반정규화가 시스템의 복잡도와 성능을 고려하여 이루어졌는지 검토. 
명칭이 같은 속성의 타입과 크기의 동일 여부 검토. ( 같은 타입을 갖도록 변경 )
내부적인 속성을 갖고 있는 속성의 존재 여부 검토. 
병합해야 할 속성의 존재 여부 검토. ( 조건에 따라 속성의 병합 여부를 결정 )
전 - 후 레코드 간 영향을 미치는 속성의 존재 여부 검토. ( 별도의 테이블 생성 )

관계 사례 분석.
개체 간 다대다 관계인 속성의 존재 여부 검토. ( 새로운 테이블을 생성해서 관계 해소 )



 # 품질 검증 결과서 작성.

품질 검증 결과서.
업무 개요 : 업무명, 업무내용, 관련 조직.
정보화 현황 : 정보 시스템명, 정보시스템 및 DB 구축 경과, 관련 지침, 정보화 과정 상 문제점.
대상기관 요구 사항 : 인터뷰 개요, 품질 현안, 진단 시 요구사항.
품질 현안 : 표준 측면, 구조 측면, 값 측면.
품질 요건 : 진단 DB특성, 진단 DB 품질 요건.
진단 방향 : 주요 품질 진단 내용 상세 기술.

품질 진단 계획서.
개요 : 품질 진단의 배경, 품질 진단의 목표, 품질 진단 내용.
품질 진단 방향 : 진단 전략, 진단 대상 DB목록.
품질 진단 방법 : 완전성, 일관성, 정합성, 업무 규칙, 품질관리수준 매핑.
품질 진단 협조 사항 : 관련 조직, 공공기관 담당자.
소요 예산 : 진단 수행에 필요한 비용.

품질 개선 계획서.
추진 개요 : 품질 개선 목적, 과업 범위, 사업 추진 일정.
품질 추진 대상 : 품질 관리 체계 수립, 표준화 수립, 데이터 보정.
품질 개선 수행 절차 : 품질 개선 컨설팅, 프로젝트 수행 절차 기술.
품질 개선 일정 및 수행 조직 : 일정, 조직 및 역할.
소요 예산 : 인력규모, 인건비, H / W, S / W, 기타 경비 등.
기대 효과 : 정량적, 정성적 분석 결과 기술.

품질 개선 결과 보고서.
추진 개요 : 품질 개선 목적, 대상 시스템 개요.
품질 오류 현황 : 품질 관리 체계, 표준화, 데이터.
개선과제 도출 : 시사점, 핵심 성공 요소, 개선 과제.
개선 방안 및 수행 : 품질 관리 체계 수립, 표준화 수립, 데이터 보정.

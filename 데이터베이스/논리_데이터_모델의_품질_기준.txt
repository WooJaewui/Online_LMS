
 논리 데이터 모델의 품질 기준.



 # 데이터 모델링의 이해.

데이터 모델링 Data Modeling.
정보시스템을 구축하기 위한 데이터 관점의 업무 분석 기법.
데이터베이스를 구축하기 위한 분석 / 설계의 과정.

데이터 모델링의 기능.
시스템을 현재 또는 원하는 모습으로 가시화하도록 지원.
시스템의 구조와 행동 명세화.
시스템을 구축하는 구조화된 틀 제공.
시스템을 구축하는 과정에서 결정한 것을 문서화.

데이터 모델링의 중요성.
파급효과( Leverage ) : 시스템 구축 중의 데이터 구조 변경 작업은 큰 위험 요소가 될 수 있음.
간결한 표현( Conciseness ) : 시스템의 정보 요구사항과 한계를 간결하게 표현할 수 있다. 
데이터 품질( Data Quality ) : 품질의 문제는 초기에는 발견되지 않고 오랜 기간 뒤에 나타난다.

데이터 모델링 3단계.
개념적 모델링 : 높은 추상화 수준. ER 다이어 그램.
논리적 모델링 : 개체, 속성, 관계, 식별자 등을 정확하게 표현.
물리적 모델링 : 실제로 데이터베이스에 이식하는 단계. ( 하드웨어 종속적 )

데이터 모델링의 3가지 개념.
개체( Things ) : 업무가 관여하는 어떤 것, 개체는 유일하게 식별할 수 있는 유일성을 가져야 함.
속성( Attributes ) : 개체가 가지는 성격. 개체 없이 단독으로 존재 불가능.
관계( Relationship ) : 업무가 관여하는 개체 간의 관계. 관계의 차수 1:1, 1:N, N:M.



 # 좋은 데이터 모델의 요소.

단순하고 명확한 모델 Elegance.
개체( 집합 )에 대한 정의가 명확해야 된다.

데이터의 무결성과 비중복성 Integrity & No Redundancy.
무결성 : 데이터베이스의 정확성을 보장하는 문제를 의미. 
데이터 무결성 보장 : 데이터 중복을 제거하는 것이다. ( = 정규화 )

비즈니스 룰 반영 & 완전성.
완전성 : 시스템의 비즈니스를 효율적이고 빠짐없이 적용.
비즈니스 룰 반영 : 업무 처리 요구사항에 만족하도록 모델링.

업무에 유연한 대처 가능 Stability & Flexibility.
업무에 유연한 대처 가능 : 확정성 있게 설계.

데이터 재사용 Data Reusability.
독립적으로 설계되어야 재사용성을 향상시킬 수 있음.

의사소통 Communication.
설계자가 정의한 업무 규칙들을 동일한 의미로 받아들이고 정보시스템을 활용해야 된다.

통합성 Integration.
전체 조직 차원에서 한번만 정의해서 참조, 활용하는 것이 좋음.



 # 품질 검증의 개요.

품질 검증.
리뷰 세션을 통해 작성된 데이터 모델의 품질을 검토.
모든 이해관계자가 동의하는 검토 기준이 필요.
'완벽한 모델' 보다 '적합한 모델' 의 관점에서 검토.

논리 데이터 모델의 품질 기준.
정확성 : 데이터 모델이 표기법에 따라 정확하게 구현되었고 요구사항이 반영되었는지 확인.
완전성 : 구성 요소를 정의하는데 있어서 누락을 최소화하고 요구사항에 있어서 누락이 없는지 확인.
준거성 : 제반 준수 요건들이 누락 없이 정확하게 준수되었음을 확인.
최신성 : 현행 시스템의 최신 상태를 반영하고 있음을 확인.
일관성 : 여러 영역에서 사용되는 데이터 요소를 한 번만 정의되고 참조 활용하는 것을 확인.
활용성 : 업무 변화시에 설계 변경이 최소화되도록 유연하게 설계되어 있음을 확인.

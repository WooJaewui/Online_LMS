 # 현행 시스템 자료수집.

UI 관련 시스템 분석.
시스템 관련 분석단계에서 도출된 '현행 시스템 분석서' 를 통해 UI 요구사항 선별 및 도출.

UI 관련 시스템 분석의 필요성.
도출되는 시스템 개발 범위를 살펴보면, 해당 시스템의 사용자가 접하는 인터페이스 유추 가능.

현행 시스템 분석서.
현재 시스템 관련 사항을 분석, 이슈를 작성한 문서.
기존 문서만으로 대신하거나 생략하기도 한다.

Front-end, Back-end 정의.
Front-end : 사용자로부터 다양한 입력을 받아 Back-end가 사용 가능한 규격을 따라 처리.
Back-end : Back-end에서 처리한 것을 Front-end를 통해 사용자에게 보여주는 역할.

클라이언트 서버 모델.
서비스 요청자인 클라이언트와 서비스 제공자인 서버 간에 작업을 분리해주는 분산 아키텍처.

3-Tire 모델.
초기 클라이언트 / 서버 아키텍처 모델은 클라이언트 계층과 서버 계층, 두 개의-Tier로 존재.
Presentation-Tier : 화면 등 사용자의 인터페이스 담당.
Business-Tier : 데이터 처리, 업무 등의 처리 담당.
Data-Tier : 데이터 저장, 검색, 관리 등을 담당.

MVC 모델.
3-Tier의 시스템 구조를 프로그래밍 개발 관점에서 서술.
Model : 데이터의 관리만을 분리하여 프로그래밍.
View : 데이터 처리, 업무 등의 처리 담당.
Control : 데이터 등의 관리 제어 부분을 프로그래밍.



 # 채널 시스템 분석.

채널 시스템.
사용자가 사용하는 디바이스를 의미한다.

채널시스템 분류.
클라이언트 채널 : 일반적인 윈도우 프로그램의 화면 UI.
웹 채널 : 웹 프로우저를 통해 조회되는 화면.
모바일 채널 : 스마트 폰, 스마트 패드 등의 화면.
기타 : ARS 시스템, 이메일 응답 시스템, 증명서 발급 시스템 화면 등.

클라이언트 채널 설계 시 고려사항.
개발 도구에 대한 성격.
화면 구성을 위해 제공되는 컴포넌트 성격.

웹 채널 설계 시 고려사항.
동적 페이지, 정적 페이지 설계.

모바일 채널 설계 시 고려사항.
모바일 기기에 따라 화면이 다르다.
PC와 다른 모바일 기기 만의 특별한 기능. ( 터치, 제스처, 센서, 확대 축소 등 )

 # UI 업무 분석 개선작업.

UI 요구사항 확인.
UI화면 설계에 영향을 미치는 요구사항 도출.
전반적 요구사항 도출 과정과 별도로 UI 설계를 위한 UI 요구사항 도출 과정을 가지기도 함.

UI 관련 업무 분석 과정.
업무 분석 프로세스를 정보시스템 구축 관점으로 크게 분석.
해당 업무에서 UI관련 업무 추출.

UI 업무 분석 및 개선 프로세스.
UI 요구사항 도출 과정은 전체 요구사항 도출하는 과정과 유사함.
실질적인 UI 업무 분석 작업은 사용자 편의성, 접근성 등 UI 설계 지침을 고려한 UI 관련 업무 개선작업.

인터뷰 조사.
화면 설계자는 기존의 업무지침, 메뉴얼 등을 분석하고, 확인이 필요한 사항을 파악.
업무 관련 담당자 및 책임자에게 인터뷰를 통해 화면 설계에 필요한 요건 사항을 도출해 냄.

역공학 분석 Reverse Engineering.
소프트웨어 시스템의 구성요소를 알아내고 분석하는 과정.
관련 서류나 자료들이 부족하거나 신뢰할 수 없는 경우 직접 소스코드를 확인하는 방법이다.

순공학 분석 Forward Engineering.
문서 산출물에서 분석되어 프로그램 코드를 생산.

업무개선 PI.
화면 UI에 영향을 미치는 업무를 개선하려는 작업의 작은 의미.



 # UI 참고 모델 분석.

UI 업무 참고 모델 Business Reference Model.
UI 업무를 분석하기 위한 참고 모델.
UI 설계 업무가 처음 수행하는 업무이거나, 기존 업무를 확인할 수 없는 경우 사용.

참고 모델 사용.
업무 참고 모델로부터 서비스가 도출되고, 서비스의 대부분은 정보시스템으로 구현되어 사용됨.
정보시스템의 세부적인 부분으로 UI 관련 업무가 도출됨.


 화면 설계 개요.



 # 응용 SW엔지니어링 수행 프로세스.

사용자 경험 디자인.
사용자가 특정한 인터랙션 시스템으로 받는 전체적인 경험을 통해 갖게 되는 긍정적 영향을 포함.

사용자 경험 디자인의 장점.
사용자의 요구를 벗어나는 요소 감소.
전체적인 사용성과 체계의 증가.
사용자 관찰을 통한 사업과 마케팅의 목표 달성.



 # 화면 설계 과정 개요.

UI 요구사항 확인하기.
관련 업무 분석.
관련 시스템 분석.
UI 요구사항 도출 및 수집.
UI 요구사항 정의 확정.
UI 요구사항 확인 및 검증.
UI 프로토타입 제작 검토.

UI 설계하기.
와이어 프레임 작성.
스토리보드 작성.
UI 흐름 설계.
UI 상세 설계.

와이어프레임 작성.
제품을 구성하는 레이아웃을 정적이고 간단한 표현 상태로 재현한 것.
간단한 모양만을 사용하여 인터페이스를 시각적으로 묘사한 것.

스토리보드 작성.
정보시스템 구축 시 각 화면의 구성 및 기능에 대해 설명해 놓은 자료를 의미.

화면 설계서 작성.
사용자가 직접 조작하는 화면에 대하여 개발자가 구현할 수 있는 수준으로 설계된 문서.

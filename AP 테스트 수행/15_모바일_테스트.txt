 # 모바일 애플리케이션 테스트 특징과 프레임워크.

모바일 애플리케이션 테스트 특징.
가상의 시뮬레이터 환경 내부에서 먼저 테스트를 함. ( Emulator )
테스트 품질이 어느 정도 완성되면 핸드폰 환경으로 넘어가서 테스트를 함.
빠른 출시 기간으로 Agile한 테스트 기법이 필요.

Agile 방법.
구체적인 계획을 세우고 순차적으로 진행하는 방식이 아닌 대략적인 계획으로
생략할 수 있는 것들은 모두 생략하면서 빠르게 애플리케이션을 완성하는 방법이다.

셀레니움 Selenium - 테스트 프레임워크.
브라우저 기반 웹사이트를 테스트하기 위해 만들어진 오픈 소스 솔루션.
RestFul API 형태.

에피움 Appium - 테스트 프레임워크.
모바일 앱 테스트를 위해 셀레니움을 확장해 개발한 테스트 자동화 프레임워크.
모든 종류의 모바일 앱 테스트 가능.
에피움 서버와 JSON Wired Protocal로 통신함.



 # 모바일 애플리케이션 테스트 도구와 고려 사항.

자동화 도구.
화이트박스 테스트 도구 : Android instrument.
블랙박스 테스트 도구 : Robotium.

SIKULI. ( 시쿨리 )
대표적인 모바일 애플리케이션 테스트 자동화 도구.
코드를 잘 몰라도 GUI 테스트를 사용해서 이해하기 쉽다.

MONKEYRUNNER ( Android 용 )
Android 장치를 원격으로 제어하는 API를 제공하는 오픈 소스 프로젝트의 일부.
동시에 여러 장치를 제어하는데 사용 가능.
애플리케이션 코드 정보가 필요함. ( GUI / IDE가 없음 )

테스트시 고려 사항.
에뮬레이터 기반 테스트 : 실제 단말이 아니라 가상의 환경에서 모바일 테스트를 함.
에뮬레이터 테스트 단점 : 실제 장치에서 테스트 시 또 다른 오류가 발생한다.
모바일 테스트는 테스트 장비와 테스터가 멀리 떨어져 있는 경우가 존재한다.

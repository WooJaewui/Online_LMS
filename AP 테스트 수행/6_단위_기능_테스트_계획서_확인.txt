 # 단위 기능 테스트 계획서 확인.

단위 기능 테스트 목적.
구현된 소스 코드를 작은 단위로 쪼개어 특정 모듈이 정확하게 동작하는지 단위 기능의 품질을 검증함.

단위 기능 테스트 범위.
개발된 시스템에 있는 프로그램.

전체적인 테스트 수행 절차.
Entry Criteria 점검 후 테스트를 수행하고 Exit Criteria 충족 후 결과 보고.
Entry Criteria : 테스트를 수행할 수 있는 상태인지 확인하는 것.
Exit Criteria : 테스트를 끝내도 되는 상태인지 확인하는 것.

결함 관리 절차.
결함 등록 -> 결함 분석 및 조치자 지정 -> 결함 Fix 및 Debugging -> Build & Deploy 
-> Re-Test 요청 -> 결함 현황 모니터링 및 이슈 관리 -> 결함현황보고.

단위 테스트 케이스 확인.
테스트 실행 이전에 어떤 단위 테스트들이 있는지 확인한다.

 # 테스트 케이스 작성법.

테스트 케이스 작성법.
SW 내부구조 참조여부 : 블랙박스, 화이트 박스.
설계근원 기준 : 명세 기반 기법, 구조 기반 기법, 경험 기반 기법.

화이트박스 테스트 작성법.
프로그램의 내부 구조를 알고 테스트 케이스를 작성하는 기법.

블랙박스 테스트 작성법.
프로그램의 내부 구조를 모르고 외부의 기능 동작만을 가지고 작성하는 기법.

명세 기반 테스트 작성법.
프로그램 요구 분석 명세서 혹은 설계사양서 등의 명세를 기반으로 테스트 케이스를 작성.

명세 기반 테스트 작성법 종류.
동등 분할 기법 : 테스트 케이스 유형을 동등하게 분할하여 테스팅.
경계값 분석 기법 : 경계 부분에 해당하는 입력값에서 결함 발생률이 높기 때문에 경계값에서 테스팅.
결정 테이블 기법 : 주요한 의사결정 요소들을 표로 만들고 요소들간의 결합에 의한 테스트 케이스 설계.
상태 전이 기법 : 이벤트, 액션, 활동, 상태 변화로 발생되는 관계 또는 동작을 파악하여 검증.
유스케이스 기법 : 유스케이스를 통해 도출되는 비즈니스 시나리오를 만들어서 테스트를 명세화.
페어와이즈 기법 : 2개의 요소의 모든 조합을 다룸.
직교 배열 기법 : 각 행과 열이 페어와이즈 하다는 것으로 조합의 수를 줄여 테스트 케이스 수를 줄임.

구조 기반 케스트 작성법.
프로그램 구조에 기반하여 테스트 케이스를 작성. ( 화이트박스와 유사하다 )

구조 기반 테스트 작성법 종류.
기본 경로 테스트 기법 : 프로그램의 기본 흐름을 테스트.
제어 구문 테스트 기법 : it, switch 같은 구문에 따라 바뀌는 경로를 테스트.
반복 구문 테스트 기법 : for-loop, while-loop 등 반복 구문에 대한 테스트.

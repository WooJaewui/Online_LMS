 # 테스트 커버리지.

테스트 커버리지.
소프트웨어 테스트 품질 측정 기준. ( 테스트의 품질 결정 )
프로그램의 테스트가 충분히 되었는가의 수행 정도를 퍼센트로 나타내는 지표.
주어진 테스트 요구사항 중 현재 테스트 과정이 달성한 요구사항을 측정한 값.

테스트 커버리지 유형.
구문 커버리지, 조건-결정 커버리지, 변경 조건-결정 커버리지, 다중 조건 커버리지.

구문 커버리지.
테스트 스위트에 실행된 구문이 몇 퍼센트인지를 측정. ( 제어문이 없다 )

결정 커버리지.
결정 포인트 내의 전체 조건식이 최소한 참-거짓 한 번의 값을 갖도록 측정.

조건 커버리지.
전체 조건식의 결과와 관계없이 각 개별 조건식이 참-거짓 한번 모두 갖도록 측정.

조건-결정 커버리지.
전체 조건식 참-거짓 한 번씩 하면서 개별 조건식 참-거짓 모두 한 번씩 갖도록 조합.

변경조건-결정 커버리지.
각 개별 조건식이 다른 개별 조건식에 무관하게 전체 조건식의 결과에 영향.

다중조건 커버리지.
결정 포인트 내의 모든 개별 조건식의 모든 가능한 논리적 조합 100% 보장.

테스트 커버리지 측정 기대 효과.
객관적인 품질 지표 생성 : 코드 커버리지를 객관적인 품질 지표로 활용.
테스트 완성도 측정 가능 : 코드 커버리지를 통해 테스트 완성도를 평가할 수 있음.
테스트 완료 시점 예측 가능 : 코드 커버리지의 적정 수준을 정하여 테스트 완료 시점으로 활용 가능.
지속적인 품질 유지 가능 : 누적 커버리지 관리 기능을 통해 변경 후 추가 테스트가 필요한 모듈 식별.
소프트웨어 품질 향상 : 테스트가 미진한 모듈을 식별하여 추가 테스트 수행 가능.

통합 테스트 이전 단계 커버리지 활용.
개발 및 단위 테스트 커버리지 측정 적용.
개발 및 단위 테스트 시점부터 테스트 커버리지 측정을 통해 충분한 테스트 케이스를 측정.

통합 테스트 시작 단계 커버리지 활용.
커버리지 측정을 통한 단위 테스트 종료 기준 적용.

통합 테스트 단계 커버리지 활용.
테스트 커버리지 누적 기능을 활용한 지속적인 커버리지 측정.
테스트 커버리지 목표 기준을 달성하기 위한 충분한 테스트 수행.

통합 테스트 완료 단계 커버리지 활용.
테스트 커버리지 목표 기준을 달성을 통한 통합 테스트 종료.

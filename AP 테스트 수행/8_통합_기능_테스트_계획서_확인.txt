 # 통합 기능 테스트 계획서 확인.

통합 테스트 목표.
사용자 요구사항 충족 및 기능 완성도 검증.
End to End 대내-외 인터페이스 검증.
데이터 정합성 및 시스템 안정성 검증.

통합 테스트 수행 절차 확인.
테스트 준비 및 진행관리 -> 테스트 수행 -> 결과 확인 
-> 결함 등록 -> 테스트 결과 검토 -> 통합테스트 시나리오 종료.

테스트 데이터 확보.
데스트 데이터 확보는 테스트 성공의 핵심요소.
테스트 장비에 데이터가 충분히 적재되었는지 확인.
민감정보는 Scrambling하여 테스트 서버에 제공.

VDI를 통한 데이터 접속.
테스터는 데이터를 입력만 가능하게 할 수 있도록 가상 컴퓨터를 사용한다. 
민감정보를 외부로 출력할 수 없도록 만드는 방법이다.

주의사항.
테스트 시작 및 종료 기준을 확인해야된다. 
Test criteria : 테스트 시작 기준.
Exit criteria : 테스트 종료 기준.

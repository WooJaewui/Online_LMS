 # 단위 기능 테스트 실행.

단위 기능 테스트 케이스 확인 순서.
단위 테스트 목록에서 단위 테스트 대상 찾기.
테스트 케이스 열어보기 및 절차 확인하기.

단위 테스트 실행 절차.
단위 테스트 케이스 확인 -> 단위 테스트 수행 
-> 단위 테스트 결함 등록 -> 결함 재 테스트 -> 단위 테스트 결과 보고.



 # 현장의 이야기.

단위 기능 테스트를 어떻게 볼 것인가.
학문 : 자바를 예를 들면 Class 하나를 단위로 생각하고 테스트를 한다.
현장 : 화면 하나를 단위로 보는 경우가 많고, 상황에 따라 단위를 다르게 설정한다.

기능 단위.
데이터 입-출력 : CRUD를 확인, 조회-항목-내용의 무결성.
화면 모듈 데이터 입-출력 : 데이터 입력과 수정 값 검증, 보고서 출력 여부, 보고서 출력 항목.
서버 모듈 : 계산 값 처리.
화면 모듈 서버 모듈 : 경고 메시지 표시.

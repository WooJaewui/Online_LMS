 # 테스트 프로세스.

테스트 프로세스.
계획 수립 단계 : 테스트 계획 수립. 테스트 시나리오 및 케이스 관리.
테스트 수행 단계 : 테스트 환경 관리. 테스트 수행 관리. 형상 및 배포 관리. 테스트 결함 관리.
테스트 결과 단계 : 테스트 결과 검토.

Entry Criteria.
테스트가 시작될 수 있는 최소한의 환경이 만족되었는지 확인하는 것.

테스트 관리자 작성.
AIT Application Integrated Test : 어플리케이션 통합 테스트.
SIT System Integrated Test : 시스템 통합 테스트.

테스트 수행 절차.
테스트 준비 및 진행 관리 -> 테스트 수행 -> 결과 확인 -> 테스트 결과 검토 
-> 성공 시 통합 테스트 종료 -> 실패 시 결함 등록 -> 테스트 다시 시행.

테스트 결함 관리 절차.
결함 등록 -> 조치팀 결정 -> 결함 분석 및 조치자 지정 -> 결함 조치 -> 해결 확인
-> Re-Test 요청 -> 처리 지연 결함 모니터링 -> Cross 결함 회의 -> 이슈 상정 -> 결함 현황 보고.

형상 관리 정의.
애플리케이션 소스 및 데이터베이스 구조를 변경했을 때 형상을 최신 버전으로 유지 관리하는 활동.

배포 관리 정의.
애플리케이션 및 데이터베이스 형상을 테스트 장비에 적용하여 테스트를 할 수 있도록 지원하는 활동.


 테스트 정의 및 테스트 표준 알아보기.



 # 테스트 개요.

소프트웨어 테스트의 정의.
테스트 계획에 따라 기능 단위가 요구 사항을 충족하는지를 점검하는 활동.

애플리케이션 테스트의 목적.
결함을 발견하고, 기능과 비기능의 적합성, 업무 처리 적절성을 보증.
기능에 구현 완전성, 요구사항 대비 기능의 정확성을 검증.
기업의 가치 혹은 이득을 극대화함. 

테스트 구성요소.
테스트 프로세스 : 정책, 전략, 관리 프로세스이 있다.
테스트 기법 : 분석 , 설계, 실행 , 자동화 기법이 있다.
테스트 조직 : 관리자, 설계자, 실행자, 개발자가 있다.
테스트 문서화 : 계획서, 결과서, 진행 보고서, 결함 목록이 있다.

테스트 정책 프로세스.
입력물 : 경영정책, IT정책.
활동 : 조직의 테스트 정책 수립, 수정.
출력물 : 조직의 테스트 정책.

테스트 전략 프로세스.
입력물 : 조직의 테스트 정책,IT 프로젝트 관리 정책.
활동 : 조직의 테스트 전략 수립, 수정.
출력물 : 조직의 테스트 전략.

테스트 관리 프로세스.
입력물 : 조직의 테스트 전략, IT 프로젝트 계획.
활동 : 프로젝트 테스트 계획 수립, 테스트 실행, 테스트 모니터링.
출력물 : 수행 계획서, 진행 보고서, 결과 보고서.

테스트 기법.
분석 기법 : 동적 분석, 정적 분석.
실행 기법 : 화이트 박스, 블랙 박스.
설계 기법 : 구조기반, 명세기반, 경험기반.
자동화 기법 : 설계 자동화, 실행-모니터링 자동화, 관리 자동화.



 # 테스트 표준.

ISO 29119. ( 국제 표준 )
소프트웨어 테스팅의 개념 및 프로세스, 원리, 가이드를 제시하는 소프트웨어 테스트 국제 표준.

ISO 29119 구성.
part1 : Concepts & Vocabulary
part2 : Processes.
part3 : Documentation.
part4 : Testing Techniques.
part5 : Keyword-Driven Testing.

IEEE 829-2008. ( 국제 표준 )
소프트웨어 테스트 문서화를 위한 표준을 제시.

IEEE 829-2008 문서의 구조 - 테스트 계획.
최상위 테스트 계획( Master Test Plan ) : 프로젝트 전체 생명주기에 걸쳐 테스트를 관리하는 문서.
단계별 테스트 계획( Level Test Plan ) : 개발 생명주기에 맞는 테스트 계획을 세우는 문서.
단계별 테스트 설계( Level Test Design ) : 테스트 항목 식별, 항목들의 성공과 실패 기준 정의.

IEEE 829-2008 문서의 구조 - 테스트 도구와 절차.
단계별 테스트 케이스( Level Test Case ) : 요구사항 대비 구현만족도를 높이기 위한 문서.
단계별 테스트 절차( Test Procedure ) : 테스트 케이스를 실행하는 순서를 의미하며 기능 평가 방법.

IEEE 829-2008 문서의 구조 - 테스트 결과와 보고서.
단계별 테스트 로그( Level Test Log ) : 테스트 활동 수행 기록 문서.
예외보고서( Anomaly Report ) : 오류 발생 시 그에 대한 기록을 적어 두는 문서.
테스트 중간 상태 보고서( Level Interim Test Status Report ) : 테스트 활동을 요약한 결과 문서.
단계별 테스트 결과 보고서( Level Test Report ) : 테스트 활동에 대한 평가를 진행하는 문서.
최종 테스트 결과 보고서( Master Test Report ) : 테스트 결과에 대한 분석과 평가를 진행하는 문서.

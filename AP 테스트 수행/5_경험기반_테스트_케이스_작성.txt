
 경험 기반 테스트 케이스 작성.



 # 경험 기반 테스트 케이스 작성법.

경험 기반 테스트 케이스 작성법.
테스터의 과거 경험, 직관, 기술 능력 등을 통해 테스트 케이스를 추출하는 기법이다.

경험 기반 테스트 작성법 종류.
탐색적 테스트 기법 : 테스트 설계, 수행 등을 정해진 시간 내에 수행하며 반복하는 테스팅 접근법.
분류 트리 기법 : 소프트웨어를 트리 구조로 분석 및 표현하여 테스트 케이스를 도축하는 기법.
체크 리스트 기법 : 평가해야 할 내용과 경험-노하우를 분류 정리하고 목록화하는 방법.
특성 테스팅 : ISO 9126 등의 품질모델에 있는 품질특성을 근간으로 테스트. ( 비기능적 테스팅에 주로 사용 )
오류 추정 기법 : 가능한 결함을 나열하고, 결함을 공격할 수 있도록 테스트 설계하는 기법.



 # 탐색적 테스트 접근법.

탐색적 테스팅 기법.
정해진 시간 내에 제품 정보 습득과 테스트 설계 및 테스트 실행을 동시에 수행하는 접근법.

탐색적 테스팅 기법의 장점.
테스트 시간이 부족할 경우 효과적으로 테스트 가능.
테스터의 경험에서 우러나오는 지식을 통하여 핵심 결함을 단기간에 발견 가능.
테스트 케이스 기반 테스트에서 발견 못하는 결함 발견 가능.

탐색적 테스팅 기법.
테스트 실행, 테스트 설계-계획, 제품 학습. ( 반복 수행 ) 

탐색적 테스팅의 특징.
테스터 개인의 지적 능력을 중요하게 여김.
테스트 실행을 중요하게 여김. ( 테스트 계획이 완벽하지 않아도 실행한 후 반복하면서 보강 )
결합 집중 원리에 충실함.
필요한 절차와 원칙이 있음.

탐색적 테스팅에 필요한 요소.
테스트 차터 : 제품의 어떤 기능을 어떻게 테스트하고 어떤 것을 중점적으로 봐야 할지 기록한 테스트 명령지.
테스트 노트 : 탐색적으로 테스팅한 기록을 남기는 수단.
타임 박싱 : 테스트 집중력을 높이기 위해 테스트 실행 시간을 제한함. ( 60 ~ 120분 )
회고 : 테스트의 결과에 대한 리뷰.

탐색적 테스트 프로세스.
계획 : 리스크 분석 및 전략 수립.
설계 및 실행 : 할당 받는 차터 테스트 수행.
종료 : 테스트 회고

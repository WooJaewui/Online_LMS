
 형상 관리 및 구성요소.



 # 형상 관리 개요.

애플리케이션 형상 관리.
애플리케이션의 변경 사항을 체계적으로 추적-통제하는 활동.
애플리케이션의 전체 생명 주기 동안에 생긴 모습을 관리한다는 의미.

애플리케이션 형상 관리 목적.
가시성 확보 : 애플리케이션의 비가시성에 의한 관리 어려움 극복.
무결성 보장 : 잦은 변경에 따른 형상물 상호 간 불일치를 추적, 통제.
병렬 개발 가능 : 다중 개발자들 간의 형상물 공유로 동시 개발 가능성을 지원.
표준 준수 : 표준 프로세스 준수.

형상 관리 구성요소.
repository : 형상을 저장하는 저장소.
CCB : 형상 관리 통제 위원회로써 변경 승인, 변경 확인 등을 관리.
CVS Concurrent Version System : 변경, 추적.
SVN Subversion : 리버전 기록 및 파일 저장, 유지.
SCI Software Configuration Item : 형상 항목.
Baseline : SCI의 하나로 기술적 통제 시점을 설정하는 형상 항목.
SCM Tool : CASE 툴로써 형상 관리의 자동화 지원.
변경 관리 프로세스 : Baseline 설정 이후의 모든 변경을 관리하는 역할.

형상 항목 관리 절차.
형상 식별 : 형상 관리 대상을 식별하고 기준선 설정 및 식별 번호 부여.
형상 통제 : 형상 변경 제안을 검토, 승인하여 반영할 수 있도록 통제.
형상 감사 : Baseline 무결성 평가 수단. 기준선 변경 시 요구사항과 일치성 여부 검토.
형상 기록 : 소프트웨어 형상 및 변경 관리에 대한 각종 수행 결과를 기록.



 # 기준선 관리.

생명주기 단계별 기준선 관리.
기능적 기준선 : 사용자 요구 기능이 정의되는 시점. ( 계획 )
분배적 기준선 : 요구 기능이 서브 시스템으로 분할되는 시점. ( 형상 통제 )
설계 기준선 : 각종 설계서. ( 설계 ) 
시험 기준선 : 시험을 위한 준비 수립 시점. ( 구현 )
제품 기준선 : 통합, 기능, 성능 등 시험 완료 시점. ( 시험 )
운영 기준선 : 개발이 완료된 후 운영 이관되는 시점. ( 설치 - 운영 )

형상 관리 조직.
분석가 : 사용자와의 협의를 통한 문제점 도출.
프로그래머 : 변경의 형태와 내용을 알아내고 실제 프로그램 수정.
형상 담당자 : 문서와 코드에 대한 변경을 계속 보관, 관리.
형상통제위원회( CCB ) : 형상 항목의 변경에 대한 평가, 조정, 승인, 기각을 결정하는 심의 조직.

형상 관리 지원 도구.
CVS : 최초의 형상 관리 도구. J빌더, 이클립스 등 다수의 개발 툴에서 직접 지원. ( 오픈 소스 )
SVN : CVS를 대체하기 위해 2000년부터 개발됨. ( 오픈 소스 )
PVCS : 써드파티 유틸리티 다수.

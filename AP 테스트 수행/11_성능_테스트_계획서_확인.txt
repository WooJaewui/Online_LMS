 # 성능 테스트 계획서 확인.

성능 테스트 목적과 범위 확인.
성능 테스트의 목적과 범위를 명확하게 이해해야 제대로 된 성능 테스트를 수행할 수 있음.

성능 테스트 수행 목적.
애플리케이션 성능 테스트 계획서에서 확인함.

애플리케이션 성능 테스트 목적.
시스템이 주어진 업무량, 응답시간, 시스템 사용률 등의 성능 목표를 달성하는지 검증.
시스템이 자원을 임계상황( 대략 60 ~ 70% )까지 사용하며 최대로 처리할 수 있는지 임계 성능 측정.
최대로 부하를 주었을 때, 시스템이 어느 정도까지 견딜 수 있는지 검증. ( 임계성능 )

성능 테스트 대상 업무 확인.
대상 업무 확인 : 어떠한 업무를 성능 테스트할 것인지 명확하게 확인.
대상 업무 화면 확인 : 성능 테스트 대상 업무에 대하여 명확하게 확인.
입력 값 출력 값 확인 : 사용자가 화면에 넣는 입력 값과 조회해서 나오는 출력 값도 같이 확인.

성능 테스트 대상 시스템 확인.
성능 테스트 대상 시스템 범위 : 성능 검증을 하는 대상 시스템을 확인.
성능 테스트 수행자 : 대상 시스템을 명확하게 인지해야함.

성능 테스트 목표 확인.
TPS( Transaction Per Second ) : 초당 시스템이 처리할 수 있는 트랜잭션의 수.
성능 테스트 수행자는 업무별 몇 TPS가 목표로 설정이 되었는지 확인.

성능 테스트 유형 확인.
단위 성능 테스트 : 화면 단위의 성능을 검증.
복합 성능 테스트 : 여러개의 단위 성능을 묶어서 한꺼번에 부하를 주입하여 검증하는 테스트.
임계 성능 테스트 : 복합 성능 테스트에서 최대의 부하를 주입하여 테스트.
연동 성능 테스트 : 타 시스템과의 연동 성능을 측정.



 # 성능 테스트 환경 확인.

Ramp Up.
부하를 주는 가상 사용자를 점차 증가시켜 목표로 하는 사용자 수에 도달하도록 하는 것.

성능 테스트 시나리오 확인.
어떤 방식으로 테스트가 진행되는지( 테스트 시나리오 ) 확인.
성능 테스트 담당자는 시나리오를 확인하면서 시스템 구조도 파악을 하고 있어야 함.

시나리오 확인 절차.
테스트 대상 -> 테스트 조건 -> 테스트 방법 -> 측정항목.

결함 관리 확인.
성능 테스트 중 발견한 결함 : 결함 관리 시스템에 등록.
시스템 환경 관련 결함 : 성능 테스트를 종료 할 수 있기 때문에 결함 관리 시스템 등록이 필수.

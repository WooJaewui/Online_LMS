 # UI 네비게이션의 이해.

UI 네비게이션 정의.
사용자가 서비스의 정보를 쉽게 접근할 수 있도록 도와주는 인터페이스.

UI 네비게이션의 필요성.
해당 서비스에 어떤 콘텐츠가 있는지 쉽게 파악.
사이트맵( 사이트 설명서 )의 역할을 하여 사이트 이용 방법을 쉽게 파악.

구성요소.
네비게이션 바 : 네비게이션 시스템의 주축이 되는 요소.
풀다운 메뉴 : 네비게이션 바와 함께 사용하는 요소.
로컬 검색엔진 : 원하는 검색어를 입력하면 사이트 내에 있는 콘텐츠 탐색.
버튼, 아이콘 : 디자인 컨셉에 맞게 제작.
사이트 맵 : 사이트 구조를 알려주는 지도.
링크 걸린 텍스트 : 텍스트를 클릭하면 텍스트에 mapping된 url주소로 이동.

UI 네비게이션 구성.
가능한 적은 용량으로 제작.
사용자들이 오래 보아도 질리지 않도록 심플하게 제작.
한 눈에 볼 수 있어야 함.
입력된 텍스트가 선명하게 보이는지 확인.
디자인 컨셉의 일관성.
현재 위치를 알 수 있도록 설계.



 # 네비게이션 디자인 및 원칙.

롤오버 메뉴.
버튼 이미지가 롤오버 되는 형태, 간결하고 제작이 쉬움.

팝업 메뉴.
상위 메뉴를 클릭하거나 마우스를 갖다 대었을 때 하위 메뉴가 나타나는 메뉴.

풀다운 메뉴.
내비게이션 바와 함께 제공되어 빠르게 레벨 이동이 용이.

이미지 맵을 이용한 메뉴.
이미지 자체에 맵을 적용해서 메뉴의 역할을 하게 만듦.

Global navigation.
사이트 전체에 걸쳐 영향력을 행사하는 네비게이션.
일관성을 유지하여 유저에게 편의를 제공함.

Local navigation.
레벨마다 다른 메뉴를 표시하여 레벨을 이동할 때 사용하는 네비게이션.

Associative navigation.
사이트에서 페이지의 위치에 상관없이 비슷한 주제나 내용을 가진 페이지들을 연결.
구조적인 경계를 넘나드는 경향이 있다. ( Link )

Contextual navigation.
임베디드 네비게이션을 포함하는데, 텍스트 자체 내에 내재되어 있어서 텍스트 링크로 표현.

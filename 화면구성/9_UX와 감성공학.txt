 # 인터랙션 디자인.

인터랙션.
사용자가 인터페이스를 통하여 시스템을 이용하는 일련의 상호작용.

인터렉션 디자인. Interaction Design IxD
서로 교류가 가능한 양방향 통신이나 서비스를 의미.
인간과 컴퓨터 상호작용을 디자인 하는 것.

대표적인 인터랙션 디자인.
애니메이션 효과, 마이크로 인터랙션.

UX, UI, IxD 관계.
UX > IxD > UI로 구성된다.

인터랙션 디자인 요소.
움직임 : 제품에 맞는 행동을 만들어 내도록 디자인 함.
공간 : 모니터나 스크린에서 3차원 공간을 잘 활용해야 함.
시간 : 인터랙션을 사용해서 사용자들에게 지연되는 느낌이 들지 않도록 고려.
외관 : 해당 서비스가 어떻게 인터랙션을 할 수 있는지 사용자에게 힌트를 줌.
질감 : 사용자에게 감성을 함께 전달하는 요소.
소리 : 소리를 다양하게 변형하여 정보를 제공.

인터랙션 디자인 접근 방법.
시스템 중심 디자인 : 시스템의 전체적인 동작 구조에 맞춰서 디자인.
기능 중심 디자인 : 시스템에 요청되는 기능의 구현에 집중.
사용자 중심 디자인 : 디자인 전 과정에 사용자를 참여시켜 검증 수행.
목표 지향 디자인 : 사용자의 목표를 찾고 그것을 중심으로 디자인.
직관적 디자인 : 디자이너의 직관과 통찰력을 통해 새로운 제품을 디자인.
천재적 디자인 : 디자이너 개인의 뛰어난 능력에 기반한 디자인.
연구 중심 디자인 : 인간과 기계의 특성을 연구하고 연구에 기반하여 디자인.



 # 프로토타입.

프로토타입의 역할.
개발 접근법의 하나.
개발 초기에 시스템의 모형을 간단히 만들어 사용자가 직접 사용해 보게 함.

프로토타이핑의 장점.
최종 사용자의 요구 극대화.
프로토타이핑을 통해 빠른 기간 안에 사용자 평가.
초기 오류 발견 및 변경의 용이.

프로토타이핑의 단점.
유지보수의 문제 발생 여지.
비용 발생.
사용자의 요구의 극대화로 효율성이 떨어지기 쉬움.

프로토타이핑의 단계.
1단계 : 사용자 요구사항 분석을 위해 시스템 설계자와 사용자 간에 함께 작업.
2단계 : 시스템 설계자는 CASE 도구를 이용하여 프로토타이핑을 개발.
3단계 : 사용자는 개발된 프로토타입을 실제 사용.
4단계 : 프로토타입의 수정과 보완 작업.

 # UI 설계 검토.

설계 개발 프로세스.
UX / UI 기획 -> UI 설계 문서 -> GUI 디자인 -> GUI 가이드라인 -> 코딩 개발.

상세 문서의 필요성.
상세 수준에 따라 그래픽 디자인과 개발 결과가 달라질 수 있다.
개발 및 출시 후 UI 의도와 결정의 근거가 된다.

디자인 문서 작성법.
화면 스케치 : 주요 화면에 대한 컨셉 스케치.
Wireframe 디자인 : 실제 적용 가능한 UI 구성요소의 적용.
화면의 상세 요소에 대한 기술 : 자세한 화면의 구성 및 요소에 대한 설명.
동작에 대한 명세 : 각각의 화면에 설명하는 구성 이외에 전체 화면 간의 흐름을 볼 수 있는 내용.

디자인 가이드라인 작성법.
최우선 목적에 집중.
최대한 간결하게 작성.
정보를 적절한 스타일로 제공, 중요한 것은 강조.
동작하는 방법을 알려주는 디자인.
인터랙션 활용.
해상도를 고려한 디자인.
Image를 활용.



 # UI 설계 피드백.

사용성 테스트 - 디자인 검증.
사용자의 니즈를 발견하는 방법. 작업의 중요성을 판단.

Heuristic Evaluation - 디자인 검증.
평가 척도에 따른 준수 여부를 사용성 전문가들이 판단하는 방법론.
장점 : 전문가들만 모여지면 언제든지 활용.
단점 : 전문가 개인의 편견이 개입될 수 있음.

Cognitive Walkthrough - 디자인 검증.
사용자 조사방법.
편리성 여부의 평가.
어떤 한 시점에서 적합한 행동을 취하는지 여부의 평가.

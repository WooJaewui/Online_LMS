
 Wireframe Mockup을 이용한 UI 설계.



 # UI 설계 용어.

Wireframe.
UI 중심의 화면 레이아웃 설계.
선을 이용하여 뼈대를 만드는 것.
디자인을 하기 전에 제작하는 화면 설계의 가장 기초적인 단계.

Wireframe을 작성하는 이유.
일정을 단축시키기 위한 방법.
제한적 사고를 타파.

Wireframe의 중요성.
기획자, 개발자 등 모든 개발 구성원들과의 의사소통.
혼자만 보는 산출물이 아니기 때문에 정확한 표현을 해야 됨.

Wireframe 작성.
Estimates ( 견적 ) : 화면 흐름에 대한 계획, 대략적이 견적.
Functionality ( 기능 ) : 특정한 엘리먼트 세트를 필요로 하는 화면들을 상세하게 리스트업.
Usability and Design draft ( 유용성과 디자인 초안 ) : 전체적인 Look을 결정하는 단계.

Mockup.
실물과 흡사한 정적인 형태의 모형.
서비스 개발 전에 결과물을 그려보는 것을 의미.

Mockup 단계의 필요성.
기획자나 디자이너의 아이디어를 그려봄으로써 서비스를 더욱 명확하게 함.

Prototype.
다양한 인터랙션이 결합되어 실제 서비스처럼 작동하는 모형.

UI 설계 도구.
Power Mockup, Balsamiq Mockup, AXURE, Kakao Oven ( 무료 )

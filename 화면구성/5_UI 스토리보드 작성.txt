
 UI 스토리보드 작성.



 # 스토리보드 이해.

스토리보드 정의.
UI / UX 구현에 필요한 사용자와 목표, 인터페이스 간 상호작용을 시각화하는 것.
개발자, 디자이너와의 의사소통을 위한 도구.

스토리보드 목적.
구현할 서비스의 필요한 기능 조작 간의 관계를 시각화하여 정리.
개발자와 사용자 간의 상호 피드백을 통해 문제점을 미리 발견하고 대처할 수 있다.

스토리보드 구성요소.
Feature List : UX 컨셉, 요구사항의 구현에 필요한 기능 리스트.
User Task-flow : 기능 수행, 사용자의 행위 매핑, 흐름도.
Information Architecture : 정보 구조화, 우선순위.
Wireframe : 인터페이스 요소의 배치, 화면구조.
Workflow : 여러 Wireframe 개별 기능 간 화면 전환 흐름도.

Feature List 작성 단계.
UI 구현에 수반되는 시스템, 서비스, 장치 등 인터페이스 Feature 목록 나열.
필요한 기능 추출 및 우선순위 정의.
사용자의 요구 사항을 충족시키기 위한 서비스의 기능 나열.
우선순위 설정 방법에 따라 기술된 기능의 우선순위 결정.
기능 간의 포함 관계를 고려하여 추가, 삭제.
정의된 기능들에 대해 우선순위 결정.

User Task-Flow 작성 단계.
기능 수행을 위한 사용자의 행위 순서( 흐름 )로 나열하고 검증하는 단계.

Process Diagram 작성 단계.
각 기능 사이 관계를 정립하여 구조화하는 단계. ( 조건과 실행으로 구분 )

Information Architecture 구조화.
각 기능 사이에 정립하여 구조화하는 단계.

Wireframe 설계.
각 기능 사이 관계를 정립하여 구조화하는 단계. 화면 단위의 레이아웃 설계.



 # 스토리보드 작성법.

표지 작성.
프로젝트 명, 문서 버전, 최종 업데이트 일자, 작성자 정보.

문서 버전 관리.
버전 관리 이력은 여러 사람이 협업하는 프로젝트에서 아주 중요하다.
변경된 문서 버전, 변경일, 변경된 위치, 변경 내용, 작성자.

목차 작성.
많은 양의 스토리보드에서 원하는 부분을 바로 찾아볼 수 있도록 정확하고 상세히 작성.

Information Architecture 작성.
서비스 메뉴의 명확한 카테고리화, 사용자 입장에서 복잡하지 않은 Depth로 작업.

공통 모듈 작성.
스토리 전반에서 매번 반복하여 작성해야 하는 내용을 공통 모듈로 작성.
네비게이션 메뉴, Header, Footer 등.

화면설계 Description 작성.
실제 구현할 화면과 화면 구성요소 등에 대해 번호를 부여하여 화면 우측에서 설명.

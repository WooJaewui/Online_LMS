 # 대외 시스템 연동의 개요.

개념.
기업 내부( 대내 시스템 )와 기업 외부의 시스템 간 연계를 의미함.

원칙.
대외 연동 : 사내망이 아닌 전용선 망을 통한 인터페이스 표준을 수립하여 적용.
직접 연동 : 모든 경유는 ESB를 통한 연동을 준수, 직접 DB 연동 패턴도 수용.
인터페이스 표준화 : 대외 공통으로 인터페이스 ID 및 표준화 체계를 통해 항목을 관리함.

기능.
연동 표준 : 연동 시스템 간 송수신 필수 항목 및 거리 패턴을 정의하는데 초점을 둠.
에러 처리 및 로깅 : Error Handling 및 로깅 기능을 제공해야 함.
스케쥴링 : 솔루션에서 제공하는 스케쥴링 기능을 사용함.
분기 로직 : 시스템 API를 이용하여 각 단위 시스템으로 분기함.
파일 연동 : FIN파일을 이용하여 업무 파일을 송/수신함.

비기능.
보안 : 대외기관에서 암호화 모듈 제공.
라우팅 설정 : 데이터에 따라 특정 시스템으로 전송이 가능해야함.
Rule 기반 전문 변환 : 자동으로 전문 변환이 가능해야 함.
Fail-over : Backup Endpoint를 판단하여 Fail-over를 제공해야함.
Thread Pool : 최소개수를 지정해서 시스템 전체에 영향을 주지 않도록 함.
테스트 : Test할 수 있어야 함.

개발 절차.
시스템 구성 -> 거래 정의 -> 운영 설정 -> 모니터링.

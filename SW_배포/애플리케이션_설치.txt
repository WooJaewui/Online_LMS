
 애플리케이션 설치 - SW 배포.



 # 주요 용어 정의.

용어.
SW 발주서 : 사용자가 특정 SW를 선택 후 이를 도입하겠다는 의사를 밝히는 문서.
인수 Test : SW가 사용자 환경에서 실제 운영될 수 있는지를 확인하는 최종 Test 단계.
설치 준비 : SW를 제한된 시간에 안정적으로 설치하기 위한 사전 준비 활동 일체.
SW 검증 : SW를 요구한 대로 개발 동작하는지를 확인하는 방법.
SW Test : 기능 및 비기능을 목표대로 달성하였는지를 확인하는 과정.



 # 애플리케이션 선택 & 전달.

발주 명세서.
사용자가 특정 SW를 선택 후 도입하겠다는 의사를 밝히는 문서.
사용자의 운영체제, 연동 시스템, 서비스 등의 다양한 제약사항을 포함.

SW 발주서 내용.
SW 명칭 : 사용자가 요청하는 명칭.
운영 환경 : 사용자가 선정한 SW 운영체제.
구성 내역 : SW를 구성하는 물리적 논리적 형태.
설치 일자 : SW를 설치할 수 있는 일자 및 시간.
설치 장소 : SW를 설치하기 위한 장소.
제약 사항 : SW 전달, 설치, 확인 등에 대한 요구사항.

SW 선정 및 전달.
SW 발주서 확인 -> 담당자 확인 -> SW 선택 -> 발주 내역 확인 -> 검증 체계 준비 -> SW 전달 -> SW 설치.



 # 애플리케이션 설치.

설치 환경 확인. ( 문서 )
SW 발주서 RFP 등의 문서상에 기술된 내용과 SW 설치 현장의 Gap 확인.

설치 환경 확인. 
전력 확보 : 설치 대상 HW 대비 전력 부족, 전력 배선 부족.
NW 환경 : NW switch의 물리적 위치, NW Cable 부족.
물리적 공간 : Server Platform 설치 공간의 부족, 장비 진입 불가.
기타 : 온도, 습도, 진동, 전자파 확인.

설치 시나리오 수립.
SW를 설치하기 위한 물리적, 논리적인 설치 순서를 결정.
HW 설치 후 SW 설치.



 # 애플리케이션 설치 확인.

인수 Test 준비.
인수 Test 시나리오 준비 -> 인수 Test 항목 조정 -> 인수 Test 판정 기준 조정 
-> 인수 Test 담당자 결정 -> 인수 Test 업무 배정.

인수 Test 시행.
예의범절 준수 : 긴장된 상황 속에서 SW 설치 진행.
절차에 따른 진행 : 계획된 순서에 의해 업무 진행.
시간 엄수 : 사전에 약속된 시간에 시행.
책임자 배석 : 설치에 대한 총괄 책임자의 배석.
사전 준비 : 담당자 동의 하에 미리 준비.

인수 Test 결과 보고.
인수 Test 결과 취합 : 개별 인수 Test 결과 취합.
인수 Test 결과 보고 : 최종 보고, 요약.
향후 Task 확인 : 영업 Process 진행.
대안 도출 : 추가 검증 방법 결정.

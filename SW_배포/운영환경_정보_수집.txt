
 운영환경 정보 수집 - SW 배포.



 # 주요 용어 정의.

용어.
운영환경 : SW가 실행되는 환경과 관리되는 일체의 행정적, 법적 실행 및 사용자 환경 일체.
인터뷰 : 사용자의 요구 및 개선사항을 수집하기 위한 활동 중 하나.
운영 : 사용자 및 관리자가 목표한 기능이 정상 제공되도록 관리하는 활동 일체.
RFP( Request For Proposal ) : 제안 요청서.
설치 : SW를 사용자 환경에서 실행할 수 있도록 배치하는 활동 일체.



 # 운영환경 정보 수집 개요.

운영환경 정보 수집 정의.
사용자 환경에 적합한 SW를 결정하기 위한 운영환경에 대한 정보를 수집하는 행위.
물리적, 논리적, 시간적, 관습적 정보를 포함.
주로 영업, 기술지원 담당자에 의해 수행되며 다양한 환경 정보와 Test 정보를 포함.
SW Package가 운영체제 등 조건에 따라 다양하게 세분화 된다.

운영환경 정보 종류.
활용기관 : 기관, 부서 특징.
법령 : 적용 기관에 적용되는 법, 시행규칙, 가이드 등.
운영체제 : OS의 종류 및 버전.
SW 종류 : 데이터베이스, 웹 서버 종류 등의 연동 SW.
성능 : NW, 시스템 성능.
사용자 : 사용자의 업무적 지식, 숙련도 등.
보안강도 : 제공되는 보안 기능의 범위.

운영환경 정보 구성.
일반 정보 : 발주 기관, 요구 기능, 요구 비기능, 프로젝트 관리 요구 사항 등.
상세 정보 : 법령, 기능 범위, 옵션 등.
부가 정보 : 제약사항, 확장 가능성, 단계별 일정.



 # 운영환경에 대한 이해관계자.

이해관계자 범위.
발주 책임자 : SW 도입에 대한 최종 결정권자, 총괄 책임자.
사업 담당자 : SW 도입을 위한 발주 ~ 인수까지의 진행 담당자. ( 실무자 )
관리자 : SW 도입 완료 후 SW 운영 관리자.
사용자 : 실제 SW를 활용하는 사용자.
검수 담당자 : 동비 대상 SW의 적절성 평가. ( 없는 경우도 많다 )

이해관계자별 요구사항.
발주 책임자 : SW 도입 효과 확인, 일정 준수.
사업 담당자 : 일정 준수, 요구사항의 반영.
관리자 : 안정적 운영, 장애 발생 최소화.
사용자 : 사업 일정, 교육 대상 및 장소.
검수 담당자 : 체계적 산출물 작성, 목표 요구사항의 준수.

운영환경 정보 수집시 주의사항.
예의 범절, 사전 예고, 일정 준수, 범위 준수, 결과 공유.



 # 운영환경 정보 수집 Process.

운영환경 정보 수집 정보.
제안요청서( RFP - Request For Proposal ) : 발주 기관, 요구 기능, 요구 비기능, 프로젝트 관리 요구사항.
SW 개발 산출물 : 지원 운영체제 종류, 목표 기능, 보장 비기능, 제약 사항.
기타 정보 : 문서화 된 정보외에 사용자 인터뷰, 설문조사 등을 통해 얻은 정보들.

운영환경 정보 수집 절차.
정보 수집( RFP, 개발 산출물 등 ) -> 영업 기술지원 담당자 -> 작성 -> 납품 확인서 작성 -> 프로젝트 책임자 확인.

정보 수집 실무.
정보 수집, 대상 확인 -> 사업 담당자 확인 -> 이해 관계자 대상 확인 -> 일정 확인 -> 인터뷰 -> 보고서 작성.
인터뷰 때는 사용자의 사례 준비를 해서 정보 수집에 이해를 돕는다. 

수집 정보 정리.
장비 위치, 설치 일정, 인수 Test( 총 Test 중에 몇 개만 뽑아서 Test )
기타 : 담당자 대기, 사전 준비.

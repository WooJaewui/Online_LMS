
 SW 배포 - SW 배포 체계.



 # 주요 용어 정의.

용어.
배포 직무 : 사용자가 요구한 SW를 안전하게 전달, 설치하는 직무.
배포 체계 : Package 생성, 저장, 전달 등의 전체 과정( Process ) 포함.
통합 배포 시스템 : 배포 과정에서 발생되는 위험을 축소하고 업무 Load를 감소시키는 역할.
형상관리 : SW에 대한 소스 코드의 버전 관리하는 활동.
CI( Continuous Integration ) : 주기마다 자동적으로 소스 코드를 다운로드하여 항상 최신 SW Build.



 # SW 배포 개요.

SW 배포.
사용자가 요청한 SW를 안전하게 전달하는 직무.
고객의 SW 수신 확인 뿐만 아니라, 설치 동작까지 확인.

SW 배포 과정.



배포 과정상의 위험.
SW 운영 환경 점검 : 운영체제 종류, Version, Platform 종류 등.
SW 부재 : 납기 시점의 해당 SW의 부재.
SW 오류 검증 :  Build 된 SW의 미검증.
분실 : 전달 Channel이나 저장매체의 분실.
무결성 훼손 : 내 - 외부적 요인에 의한 무결성 손상.
기밀성 유출 : SW 자체에 대한 기밀성 훼손.
기타 : 설치 오류, 운영 오류, 법령 위반 등.



 # SW 배포 체계 수립.

SW 배포 체계의 개념 및 목적.
사용자가 요청한 SW를 안전하게 전달하는 방법을 의미.
배포 조직의 구성, 절차, 산출문, 전담 인력, 후속 관리 등의 일련의 전체 Process를 포함.

SW 배포 산출물.
RFP( Request for Proposal ) : SW 개발 , 납품 , 유지보수 등에 대한 전반적 사항을 기술. ( 제안 요청서 )
제안서 : REF에 따른 Project 수행에 대한 세부 대응 방안.
SW 메뉴얼 : SW 제품에 대한 설치, 관리자 설명서.
설치 계획서 : SW를 설치하기 위한 사전 준비 문서.
인수 확인서 : 고객사 시스템에 설치 및 동작 여부를 확인한 문ㅁ서.



 # 통합 배포 시스템.

통합 배포 시스템 역할.
SW를 배포하는 과정의 다양한 위험을 축소.
배포 담당자의 업무적 Load를 축소.
배포 과정상에서 실수, 오류, 사고 발생 확률을 최소화.

통합 배포 시스템 제공 기능.
CI ( Continuous Integration ) : Daily Build.
Version 관리 : SW Package에 대한 Version관리 . 
소스 코드 검사 : 개발 규정, 코딩 규칙, 저작권 침해 등의 지침 준수 여부 확인. 
Test : 테스트 자동화에 따른 Build Test의 정적 - 동적 테스트 수행.

통합 배포 시스템 종류.
OSS Open Source Software : Jenkins, TeamCity. ( 오픈 소스 소프트웨어 ) 
상용 : JIRA, TFS( MS Team Foundation Server ), BamBoo.

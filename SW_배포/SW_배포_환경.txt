
 SW배포 - SW 배포 환경.



 # 주요 용어 정의.

용어.
Build Number : 특정 SW Package에 대한 Build Index, Build 횟수나 일자, 시간 등을 활용.
Revision Number : 개발 진행 정도, 개발 성숙도, Package 연관 정보 등을 나타내는 지표.
배포 도구 : 형상관리 시스템, Build 서버, Build 관리 시스템, Testbed, Package관리시스템 등.



 # SW 배포 이해 관계자.

이해 관계자.
영업 : SW 사용 고객 발굴, Life-cycle Monitoring.
기술지원 : SW 고객 전달, 장애 대응, 요구사항 수집.
개발 : SW 개발.
서비스 데스크 : 기술지원 / 교육접수. 

이해 관계자 요구사항.
영업 : 지원 범위의 확장, 기능 확대.
기술지원 : 기능 / 성능에 대한 보장.
개발 : 정확한 요구사항, Scenario의 명확하.
서비스 데스크 : 서비스 안정, 효율적 고객 지원.
공통사항 : 안정성, 일정 준수.



 # 형상 관리.

형상관리 정의.
보통 소프트웨어 소스코드의 버전관리를 의미한다.
SW개발의 가시성과 추적을 위해 소스코드를 관리하는 체계.
소프트웨어를 이루는 소스코드의 Baseline을 정하고 변경을 통제하는 시스템.

설명.
보관, 접근의 문제 : 개발자 PC 내 소스 코드 보관의 위험, 비인가 반출.
동시 변경 & 충돌 : 동일 소스 코드를 동시 사용, 변경 내용 Update 필요.
백업 / 복구 : 시스템 장애 발생으로 인하여 저장된 소스 코드 삭제.
Branch : 다양한 요구사항 충족에 의한 Version 다양화.

형상관리 시스템의 제공 기능.
Upload( import ) : 소스 코드를 등록.
Export : 마지막 Revision에 대한 소스를 개발자 PC로 다운로드.
Check-out : 저장소 내의 특정 프로젝트 또는 소스 코드를 Local로 다운로드.
Commit( Check-in ) : 작업 디렉토리 변경 사항을 저장소에 적용.
Update : 저장소 내의 변경사항을 개발자 PC에 적용.
Compare : 2개의 Revision에 대한 차이점 비교.
Review Log : 특정 revision에 대한 Log 조회.
Tag : 특정 Revision을 포함한 전체 소스 코드에 대해 Branch를 생성.

정리.
형상관리 : SW의 Life - Cycle을 관리.



 # SW 빌드,  배포 도구.

배포 도구.
형상관리 시스템 : 소스코드 관리.
Build 서버( 컴파일 서버 ) : 형상관리 시스템의 소스코드를 다운받아서 컴파일.
Build 관리 시스템 : 빌드를 어떤 방식으로 할 것인지 결정.
Testbed : Build 서버에서 컴파일 한 소스를 테스트.
Package 관리 서버 : 성공적으로 테스트 된 소스코드를 패키지로 저장. 
ITSM( IT Service Management ) : 고객에게 제공하는 정보 기술 서비스를 위해 수행되는 활동 전반을 의미.

배포 도구 활용 시 주의사항.
소스코드 버전 - 패키지 버전 - 고객사 시스템의 SW관련 프로그램들이 모두 일치해야된다. ( history 관리 )

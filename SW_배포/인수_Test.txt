
 인수 Test - SW 배포.



 # 주요 용어 정리.

용어.
인수 Test : 사용자 환경에 요구한 SW 설치 정상 여부 확인, 검증 행위.
배포 위협 : SW 배포 과정상에는 준비, 전달, 설치 과정 등의 다양한 위협이 존재.
Big-Bang Test : 전체 기능을 통합하는 Long Scenario 기반 Test.
전수 Test : 개발 단계에서 시행된 시험을 모두 수행.
Sampling Test : 개발 단계에서 활용한 Test 항목 중 일부 선택 방식.



 # 배포 과정상의 위협.

배포 과정상의 위협.
불안전한 정보 : 필요한 Data중에 빠진 Data가 있는 경우.
잘못된 정보 : 필요한 Data는 모두 있지만, 잘못된 Data가 있는 경우.
조사 대상 부재 또는 오선정.

배포 과정상의 위협을 줄이는 방법.
RFP( 제안 요청서 ), SW 개발 산출물 등의 서류들을 찾아서 확인해 본다.

전달 과정상의 위협.
SW 분실 : 설치 Package의 분실.
Package 손상 : 정전기, 진동 등에 의한 SW/HW 손상.
버전 부적절 : 잘못된 정보에 의한 Package 오선택.
NW 사용 불가 : 폐쇄망에 따른 온라인 설치 불가.

설치 과정상의 위협.
Package 오류 : Pacakge 배치시 발생.
여유 공간 부족 : Package 설치 시작시 발생.
필수 Library 부재 : 설치 진행시 발생.
시스템 Panic 발생 : 설치 종료 후 발생.



 # 인수 Test 개요.

인수 Test 정의.
사용자 환경에 따른 SW 설치 정상 여부 확인, 검증 행위.
SW 특성에 따라 상이한 인수 Test를 수행.
사용자가 직접 하는 경우가 많다. 

인수 Test와 설치 후 검수 Test 차이점.
인수 Test : 실제 사용자가 사용하면서 시행하는 Test.
검수 Test : 사용자에게 전달하기 전에 하는 시행하는 Test.

인수 Test 시행 목적.
요구사항 검증 : 사용자 입증에서의 요구 검증.
운영환경 적응도 평가 : 개발, 배포 환경 대비 운영환경에 대한 적응 수준 확인.
돌발 상황 확인 : 현장에서 발생할 수 있는 문제점 조사.
개선사항 도출 : 문제점 확인 시 개선사항 확인, 분석.

인수 Test 시행 시 주의사항.
사전 준비 : 인수 Test 일정, 시간, 대상 결정.
만족 기준 수립 : 인수 여부 확인을 위한 만족 기준 사전 수립.
범위 확대 : 요구사항 대비 발생 가능성 위험 분석, 대비.
조사 분석 : 인수 Test 시행 중 발생하는 문제점 Check 및 분석.
Test 결과 공유 : 시행된 인수 Test 결과 공유, 향후 대책 수립.



 # 인수 Test 방법 및 절차.

인수 Test 기법 - Big Bang Test.
전체 기능을 통합하는 Long Scenario 기반 Test.
중요 기능을 중심으로 Test 수행.
높은 효율성, Minor 기능 확인 불가.

인수 Test 기법 - Sampling Test.
개발 단계에서 활용한 Test 항목 중 일부 선택 방식.
중요도, Random, 시험 시간 기반으로 선택 가능.
중요 기능 확인 불가 가능성이 존재.

인수 Test 기법 - 전수 Test.
개발 단계에서 시행된 시험을 모두 수행.
기능, 비기능 시험을 모두 수행.
높은 신뢰도, Test 기간 장기화.

인수 Test 절차.
설치 일정 협의 -> 시험 항목 협의 -> SW 설치 -> 검수 Test -> 인수 Test 시행 -> Test 결과 보고.
Test 결과가 좋지 않으면 앞부분부터 다시 수행.

인수 Test 결과 보고.
SW 설명 -> 인수 Test 범위 설명 -> Test 결과 -> 세부 내역 -> 조치 방법 협의 -> 향후 일정 수립.
